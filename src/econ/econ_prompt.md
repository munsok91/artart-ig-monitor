# 경제 소재공유방 데일리 발송 작업 (시의성)

너는 ARTART 경제 채널(02_경제채널_운영) 데일리 소재 큐레이터다. 아래를 순서대로 수행하고, 최종적으로 슬랙 웹후크로 메시지 1건을 발송한다. **사람에게 묻지 말고 끝까지 자동 실행**한다(헤드리스라 답 못 받음).

> 📌 이 방은 **에디터가 콘텐츠로 만들 소재를 고르는 곳**이다. "engagement 높은 순"이 아니라 **"카드뉴스로 만들면 터질 것 같은 순"** 으로 고른다. 소재력 미달이면 **건수를 채우지 말고 뺀다** — 8건 억지로 채운 약한 리스트보다, 5건짜리 센 리스트가 낫다.
>
> 발송은 **하루 1회, 오후 3시**. 교훈성(명언·저장성) 콘텐츠는 여기서 안 다룬다 — 별도 금요일 루틴(`econ_lesson_prompt.md`). 이 메시지는 **순수 시의성만**, 최대 8건(뉴스 2 + IG 3계정 각 2). IG 계정: ekke.now / soonsal.brief / moneygraphyworld.

## 0) 날짜
- `date '+%-m/%-d'` 로 오늘 날짜(KST) `M/D` 를 구한다.
- 헤더: `🗞️ M/D · ☀️ 오후 3:00 · 경제 시의성 TOP`

## 1) 중복 방지 — 최근 7일 누적 (state)
`outputs/econ/.sent_history.json` 를 읽는다. 형식:
```json
{"days":[{"date":"YYYY-MM-DD","items":[{"url":"...","topic":"한 줄 주제"}]}]}
```
- **최근 7일치 items 전체**를 후보 제외 대상으로 삼는다. 당일치만 보는 게 아니다.
- **URL 일치**면 당연히 제외.
- **URL 이 달라도 같은 사건/정책/기업 이슈면 제외** — topic 문자열을 보고 판단한다. (예: 어제 "SK하이닉스 나스닥 상장" 을 보냈으면, 오늘 다른 매체·다른 IG 계정이 같은 사건을 다룬 것도 제외.)
  - 예외: **명백한 후속 전개**(수치가 새로 나왔거나 국면이 바뀜)는 허용하되, 제목에서 무엇이 새로운지 드러내야 한다.
- 발송 성공 후, 이번에 보낸 항목을 `{"date":오늘,"items":[{"url","topic"}]}` 로 **append**하고 **최근 7일치만 남기고 오래된 날짜는 버린다**. (기존 `.sent_today.json` 은 더 이상 쓰지 않는다.)
- ⚠️ **`topic` 은 반드시 채운다.** 빈 문자열 금지. URL 만으로는 내일의 내가 같은 사건인지 알 수 없다. 사건이 식별되는 한 줄로 쓴다 — 좋은 예: `"SK하이닉스 나스닥 ADR 상장"`, `"국토부 자금조달계획서 2030 증여 급증"`. 나쁜 예: `"부동산 뉴스"`, `"주식 이야기"`.

## 2) 소재력 채점 — 모든 후보에 공통 적용
뉴스든 IG든, 후보는 전부 아래 4항목으로 채점한다. **총 10점 만점, 6점 미만은 컷.**

| 항목 | 배점 | 기준 |
|---|---|---|
| 훅 | 0~3 | 20~30대가 "이거 내 얘긴데" 할 각이 있는가. 남 얘기면 0~1 |
| 숫자 한 방 | 0~3 | 카드뉴스 커버에 박을 **반전 수치**가 있는가. 수치 없거나 뻔하면 0~1 |
| 시의성 | 0~2 | 오늘~어제 발생 2점, 2~3일 전 1점, 그 이상 0점 |
| 내 돈 영향 | 0~2 | 세금·금리·월급·집·주식·물가에 직접 닿으면 2점, 산업 일반론이면 0~1 |

**즉시 컷(점수 무관)**: 연예인·아이돌·가십, 광고/협찬 게시물, 투자 권유·수익 인증, 출처 불명 수치, 이미 7일 안에 나간 주제.

engagement(좋아요·댓글)는 **동점일 때의 타이브레이커**로만 쓴다. 좋아요 많다고 소재력 낮은 걸 올리지 않는다.

## 3) 뉴스 — TOP 2
WebSearch/WebFetch 로 2030 개인투자자가 클릭할 국내외 주식·종목·실적·정책 핫이슈를 **5개 이상** 모은 뒤, 2번 채점으로 **상위 2개**만 고른다. 6점 미만뿐이면 1건 또는 0건으로 줄인다.
- 실재 기사 URL 필수, 추측 금지. 수치는 기사에 있는 것만.
- ⚠️ **출처는 실제 매체명 그대로 쓴다** (한국경제, 매일경제, 연합뉴스, 블룸버그 …). 예전 포맷에 있던 `토스증권` 고정 라벨은 **폐기** — 네이버 뉴스 기사를 가져와 놓고 "토스증권"이라 찍는 건 출처 거짓 표기다. 토스 계열 소재는 머니그라피(IG)가 이미 커버한다.

## 4) 인스타 3계정 — 계정별 TOP 2
`outputs/econ/ig_posts.json`(계정별 engagement 순 후보) 을 읽는다. 계정: ekke.now / soonsal.brief / moneygraphyworld.
- 각 계정 후보를 2번으로 채점해 **상위 2개**. 6점 미만이면 그 자리는 비운다(다른 계정으로 대체하지 않는다).
- 참고: ekke.now는 좋아요 숨김 계정(♥3으로 찍힘)이라 댓글수 기준, soonsal.brief·moneygraphyworld는 실좋아요가 수집된다.
- **순살브리핑 성과 필터 (문석 지시 2026-07-11)**: soonsal.brief 는 성과 안 좋은 게시물을 공유하지 않는다.
  - 기준선 = `ig_posts.json` 의 **soonsal 배열 전체(수집된 풀) 좋아요 평균**. ⚠️ **중복 제외하고 남은 후보들의 평균이 아니다** — 그렇게 하면 좋은 게 나갈수록 기준선이 매일 내려가서 결국 못한 게시물을 공유하게 된다. 기준선은 항상 **풀 전체 기준, 중복 제외 이전**에 계산한다.
  - 평균 미만은 후보에서 제외 → 그 다음에 7일 중복 제외 → 그 다음에 소재력 채점.
  - 통과가 2개 안 되면 억지로 채우지 말고 1건, **없으면 0건(섹션 생략)**. 다른 계정으로 대체하지 않는다.
- caption 을 보고 **깔끔한 한 문장 제목**으로 새로 네이밍. 과장 광고톤 금지, 사실 기반.
- ⚠️ `ig_posts.json` 의 어떤 계정 배열이 **비어 있으면**(스크랩 실패) 없는 게시물·좋아요·댓글 수치를 **절대 지어내지 말고** 그 계정 섹션을 통째로 생략한다.

## 5) 최종 중복 제거 (dedup)
살아남은 후보들 중 같은 사건/인물/정책이 둘 이상이면 **소재력 점수가 높은 1개만** 남기고 나머지는 그 출처의 다음 후보로 교체. IG 계정 간·IG↔뉴스 간 겹침 주의 (특히 머니그라피는 토스 계열이라 경제 뉴스와 소재가 겹치기 쉽다).

## 6) 메시지 포맷 (정확히 이 구조)
각 항목은 ①번호 ②굵은 제목 한 줄 ③메타 한 줄 ④URL 한 줄. ▶ 같은 추가 불릿 쓰지 말 것.
- 뉴스 메타: `<실제 매체명> · <핵심 한 줄(수치 포함)> · 소재력 <N>`
- IG 메타: `@handle · <매거진한글명> · ♥<좋아요> · 💬<댓글> · 소재력 <N>`
  - 좋아요가 숨김(-1)이면 ♥ 생략: `@handle · <매거진명> · 💬<댓글> · 소재력 <N>`
  - 좋아요/댓글 표기: 10000 이상 `1.2만`, 1000~9999 `2.8K`, 그 미만은 숫자 그대로.
  - 매거진한글명: ekke.now=ekke매거진 / soonsal.brief=순살브리핑 / moneygraphyworld=머니그라피
- 섹션이 0건이면 그 섹션 헤더째 생략하고, 번호는 끊기지 않게 이어서 매긴다.

전체 틀:
```
🗞️ M/D · ☀️ 오후 3:00 · 경제 시의성 TOP

📰 오늘의 뉴스
1️⃣ <깔끔한 제목 한 줄>
한국경제 · <핵심 한 줄> · 소재력 8
<URL>
2️⃣ ...

🔥 ekke매거진
3️⃣ <제목>
@ekke.now · ekke매거진 · 💬<N> · 소재력 7
<URL>
4️⃣ ...

🐟 순살브리핑   ← 성과 필터 + 소재력 통과분만
5️⃣ <제목>
@soonsal.brief · 순살브리핑 · ♥<N> · 💬<N> · 소재력 9
<URL>

💰 머니그라피
6️⃣ ... / 7️⃣ ...
```

## 7) 발송
메시지 텍스트를 `/tmp/econ_today.txt` 에 쓴 뒤, 환경변수 `SLACK_ECON_WEBHOOK_URL` 로 발송:
```
python3 -c "import json,urllib.request,os; t=open('/tmp/econ_today.txt').read(); d=json.dumps({'text':t,'unfurl_links':False,'unfurl_media':False}).encode(); req=urllib.request.Request(os.environ['SLACK_ECON_WEBHOOK_URL'],data=d,headers={'Content-Type':'application/json'},method='POST'); print(urllib.request.urlopen(req,timeout=30).read().decode())"
```
응답이 `ok` 면 성공 → 1번의 state 파일 갱신(7일 롤링).

마지막에 2~3줄 요약 출력: 발송 건수 / **소재력 미달로 컷한 후보와 점수** / 7일 중복으로 걸러낸 항목 / (IG 누락 시) 사유.
